# 环境分析

量潮科技当前所处的结构性环境。

## 总体处境

量潮处于**"微型巨兽的悖论"**——业务在解决只有规模化公司才会遇到的复杂问题（战略早熟性），但资源远未达到规模化公司的水平（资源滞后性）。旧引擎（创始人个人资产的生命周期）已到极限，必须更换为新引擎。

具体表现：有产品无渠道，获客受制于市场信任水平低。

三条线之间存在一个因果链：无渠道所以无法持续展示能力，无法展示所以信任建立不起来，没有信任所以客户不敢下单。反之，案例库、销售链路、内容输出都是在打破"没证据→不信任→不下单→没证据"的循环。

这个问题的根因不全是渠道策略，还有一层时间维度：当前阶段的核心工作是萃取——把创始人的经验搬出来变成可见的结构。产品描述还不够清晰时，渠道无从铺起。无渠道不是因为渠道策略错了，而是因为萃取还没完成。当描述从"我在做什么"变成"我提供什么"时，渠道自然有东西可推。

本周的战略推演进一步定位了萃取的方向——**认知操作系统**。互动剧本模式证明了这一萃取路径的可行性：将创始人的战略判断和经验，转化为可被专业团队理解和执行的系统。

## 内部环境

### 各业务线的环境位置

- 量潮数据：已验证交付能力，需突破获客瓶颈。关键约束是缺少公开案例库，潜在客户看不到交付证据。新方案：动态蓝图模式降低需求沟通摩擦
- **量潮课堂：已找到PMF**（互动剧本模式），从"商业模式待跑通"进入"执行落地"阶段。关键约束从方向不清转向创始人执行瓶颈（没有时间推）
- 量潮咨询：刚完成获客模式重构。从依赖合作方转向自主获客，正处于积累期
- 量潮云：从内部平台走向开源开放。重构进度慢，需要外部贡献者跟上信息流

### 核心结构矛盾

#### 服务与产品的双线拉扯

公司处于"从项目制服务转型产品制（PaaS 云厂商）"的过渡期。不是初创公司找 PMF，而是已运营多年的业务体系在重构底层模式。

- 项目是现金流和案例来源，但项目制的边际成本不降
- 产品是终局方向，但研发投入大、周期长、短期内不直接创收
- 两条线在争夺同一批人的时间和精力
财务压力迫使项目线权重上升，产品转型节奏被进一步拉长

#### 多业务线与资源稀缺

四条业务线（数据、课堂、咨询、云）并行，各有独立的 PMF 方向和增长逻辑，但所有线共享同一组人力池和认知带宽。

核心表现：每条线都知道自己该往哪走，但都没有足够的专门资源去走。

#### 创始人认知瓶颈

创始人同时承担战略判断、产品设计、技术决策、招聘把关、品牌策略的角色。感受到"一盘大棋"的内在关联，但大脑已无法独自处理全部信息——第二大脑建设是对此的结构性回应。

当前瓶颈的深层表现不仅是认知负载过高，而是存在一个三层断层：创始人的认知层次 > 第二大脑的建设阶段 > 团队的信息处理能力。第二大脑帮助有限，部分原因在于它需要团队共同使用才能发挥效用，但团队尚未达到这个层次。结果是创始人逃不出具体决策，形成"认知越高—介入越深—团队越依赖"的循环。

本周推演发现了一个可能的突破方向：**复刻路径**——让团队走一遍创始人10年前从学术机构起家的商业化路径，以1-2年为周期。这为三层断层提供了一个结构性的弥合方案。

## 外部环境

### 市场端

- 客户信任：交付能力已获认可，但获客渠道单一、订单周期变长。新洞察：销售提成模式与客户心理不符，利他式商业化（用调研代替推销）更符合自身基因
- 竞争壁垒：AI 正在商品化技术执行能力。竞争优势需要重新定义——不是做得更快，而是做到 AI 做不了的事。真正的壁垒只有两种：效率优势（花钱花得更少），以及人类独有的认知和判断（花 token 也干不到的事）
- **海外科研市场**：本周发现的新机会点——国外科研客户以国外价格购买AI服务，中国团队有性价比和专业度双重优势
- 人才供给：市场上缺少符合标准的复合型人才，无法直接招聘，必须自建培养体系

### 行业趋势

- Vibe Coding / AI 辅助编程降低了代码产出门槛，但提升了验收和治理的要求
- 技术岗与非技术岗的边界正在模糊，组织需要重新定义岗位和能力模型
- 企业客户对数字化转型的需求存在，但谈判周期长、需求非标、决策链复杂
- 互动剧本式教学/需求沟通正在成为新的数字化服务范式

## 信号

### 获客与交付

- 从获客到交付的完整链路已跑通（直接对接→报价→转化→流程），可被复制
- 客户需求正在从"能不能做"转向"做过什么"，案例库成为跨业务线的通用杠杆
- 基本单元经济模型清楚：每个项目的利润空间、可出让比例（20-30%）有明确估算
- 利他式商业化（回访调研代替推销）更符合自身基因，需进一步验证

### 标准与质量

- 标准没有被稀释：招聘面试70人一个没要，评分体系80分合格线严格。压力下不降标准是稀缺的组织能力
- 开源社区对优质项目有需求，标准严格反而是资产，不是障碍
- 招聘标准正在倒逼供应链（实训基地、开源贡献者），标准和培养体系可能本身就是产品

### 组织与认知

- 四条业务线方向已收敛（数据做案例库、课堂做互动剧本、咨询做自主获客、云做开源），资源进来后不需要试错
- 咨询与课堂存在天然交叉导流，业务线之间有未开发的协同结构
- 第二大脑已实际运转，信息流已由AI维护。效果不够好但正在使用的系统，比空置的知识库好一个数量级
- 元认知能力：每日记录情境和决策，意味着不仅在经营公司，还在观察自己怎么经营
- **范式跃迁信号**：从"课程平台重启"到"认知操作系统"的推导链完整自洽，五个层次首尾相连，说明战略认知正在进入新阶段
- **元能力市场化可能**：假设验证方法论是否可以成为售前工具，是本周提出的关键杠杆，成本为零可立即实验
